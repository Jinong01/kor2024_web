[ 롬복 ]
    1. 정의 : 반복되는 코드들을 줄여주는 라이브러리
    2. 내 프로젝트의 롬복 적용/설치 하는 방법
        build.gradle 에 Lombok 의존성을 추가한다.
        인텔리제이의 Lombok 플러그인을 다운한다.

    3. 제공 : @어노테이션 이용
        1/ 생성자를 대신 만들어준다.
            디폴트 생성자 : @NoArgsConstructor
            모든 매개변수 생성자 : @AllArgsConstructor
        2/ getter and setter 메소드를 대신 만들어준다.
            @Getter
            @Setter
        3/ toString 메소드 대신 만들어준다
            @ToString
        4/ Builder 패턴을 지원한다.
            @Builder : **

    4. 빌더 패턴 : 인스턴스(객체) 만드는 패턴
        1. 정의 : 생성자의 단점을 보완 * new 키워드 없이 인스턴스 만들기
        2. 사용법 : 전제조건은 해당 클래스가 @Builder 이면 가능하다
            클래스명.builder().멤버변수명1(대입할값)


* 생성자란?
- new 키워드 뒤에 사용하면서 인스턴스(객체)를 생성할 때 멤버변수 초기화하는 메소드
- 단점 이자 생성자 사용의 규칙 :
    1.상황에 따라 초기회가 다르면 여러 생성자가 필요하다. [오버로딩]
    2. 생성자 관리 힘들다.
    3. 매개변수 순서와 개수를 인수값과 일치해야한다.
